扫描记录用于记录订单在仓库和派送过程中的所有扫描操作，方便查询包裹经过的处理节点、扫描时间及扫描账号，并协助排查包裹状态异常、批次异常或轨迹未更新等问题。通过扫描记录，用户可以确认订单是否经过分拣、是否完成预扫描、司机是否已领取包裹，以及扫描动作是否成功更新了订单轨迹。

功能路径：订单管理 → 中大件订单列表 → 订单列表详情

系统会展示当前订单所有已识别到的扫描记录
### 扫描类型与轨迹节点对应

<ZoomImage src="/Pasted image 20260721184353.png" alt="扫描类型对应" />
| 扫描类型  | 对应轨迹节点 | 更新后的订单状态 |
| ----- | ------ | -------- |
| 分拣机扫描 | 包裹分拣   | 仓库收件     |
| 人工分拣  | 包裹分拣   | 仓库收件     |
| 仓库预扫描 | 司机预扫描  | 已代扫      |
| 司机扫描  | 派送中    | 待收货      |

扫描记录仅表示系统识别到了对应的扫描动作，并不代表订单轨迹一定已经更新。

### 扫描说明
#### 仓库扫描说明
分拣机扫描和人工分拣均属于仓库扫描，因此并不一定只发生在包裹首次入仓时。

根据实际业务流程，这两种扫描可能出现在以下场景：

- 包裹首次到达仓库进行分拣；
- 包裹首次派送失败后退回仓库；
- 包裹三次派送失败后返仓；
- 包裹因异常无法派送，被仓库重新入库。

如果人工分拣发生在所有扫描记录的最后一次，并且订单已经结束派送流程，则该扫描通常表示包裹已返仓。

| 扫描类型 | 对应轨迹节点 | 更新后的订单状态 |
| --- | --- | --- |
| 人工分拣（返仓） | 包裹返仓入库 | 已返仓 |

因此，在分析扫描记录时，应结合订单当前状态及扫描时间综合判断扫描动作对应的业务场景。

#### 扫描成功判断说明
扫描记录只表示系统识别到了扫描动作。

如果由于网络异常、重复扫描或其他原因导致扫描失败，扫描记录仍可能被保存，因此仅凭扫描记录不能判断扫描是否成功。

建议同时查看订单详情页面【货物信息】列表中的【查看轨迹】功能。如果扫描记录对应的轨迹节点已经生成，并且订单状态发生更新，则说明本次扫描成功；如果扫描记录存在，但轨迹没有新增节点，则说明扫描动作未成功更新订单轨迹，需要进一步排查。

#### 空批次说明
部分订单虽然存在仓库扫描记录，但主批次或子批次仍为空。

这种情况最常见于人工分拣。如果操作人员在 App 进行人工分拣时未选择批次，系统仍会保存扫描记录，但不会将订单归入任何批次，因此订单会处于空批次状态。

可以通过以下两种方式确认：
**方法一：** 查看订单详情
在订单详情的【货物信息】标签页中查看主批次及子批次字段是否为空。

<ZoomImage src="/Pasted image 20260721190931.png" alt="批次查询" />

**方法二：** 查看人工分拣记录
进入：
> 仓库管理 → 人工分拣管理 → 人工分拣

使用订单跟踪单号进行查询。

| 建立批次 | 说明 |
| --- | --- |
| 是 | 本次人工分拣已建立或加入批次 |
| 否 | 本次人工分拣未建立批次，订单仍属于空批次 |

### 常见问题
1. 为什么有扫描记录，但订单状态没有变化？

	扫描记录表示系统识别到了扫描动作，并不代表扫描已经成功更新订单轨迹。建议结合【查看轨迹】确认是否产生了新的轨迹节点。
	

2. 只要有扫描记录就一定扫描成功了吗？

	同一个包裹在扫描失败、重复扫描或重新处理时，都可能产生多条相同类型的扫描记录，因此出现短时间内连续多次扫描属于正常情况。

3. 为什么人工分拣有时对应的是“包裹返仓入库”？

	人工分拣不仅用于首次仓库分拣，也可能用于派送失败后的返仓处理。当人工分拣发生在返仓流程中时，对应的轨迹节点会更新为“包裹返仓入库”，订单状态更新为“已返仓”。
