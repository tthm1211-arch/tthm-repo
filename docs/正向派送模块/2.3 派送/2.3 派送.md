派送是正向派送业务的最后一个环节，也是订单从仓库流转至收件人的执行阶段。

在进入派送前，订单需完成以下前置操作：

1. 在管理端完成派送规划，为订单分配对应的路线及派送司机
2. 仓库完成包裹预扫描，并根据司机及路线信息完成排单

完成以上操作后，订单将同步至司机 App，司机即可进行扫码取件、装车及派送。

派送过程中，管理端主要承担**订单监控**及**异常处理**的职责，包括实时查看订单派送状态、轨迹信息、签收证明及派送失败原因，并支持对异常订单进行人工处理，例如修改收件人信息、拦截订单及修改 POD（签收证明）等。

### 管理端相关模块
正向派送涉及的管理端模块如下：:

```
管理端
├─ 订单管理
│   └─ 中大件订单列表
│       ├─ 订单列表详情
|        │       ├─ 基本信息
|        │       ├─ 货物信息
|        │       ├─ 司机信息
|        │       ├─ 签收证明
|        │       └─ 派送失败信息
├─ 轨迹管理
│   └─ 业务轨迹绑定
│       ├─ 业务轨迹绑定配置
├─ 监控报表
│   └─ 报表统计
|        │       └─ 正向派送异常监控
```

| 模块       | 路径                     | 功能                                                 |
| -------- | ---------------------- | -------------------------------------------------- |
| 中大件订单列表  | 订单管理→ 中大件订单列表          | 正向派送业务的核心页面，用于查询订单信息、查看派送状态、轨迹、司机信息及签收证明，并支持订单相关操作 |
| 业务轨迹绑定   | 轨迹管理 → 业务轨迹绑定          | 配置及维护业务轨迹状态机，定义订单状态与轨迹节点之间的对应关系，以及各轨迹节点的流转规则       |
| 正向派送异常监控 | 监控报表 → 报表统计 → 正向派送异常监控 | 用于监控正向派送过程中的异常订单，帮助运营人员及时发现并处理异常情况 0..0            |
