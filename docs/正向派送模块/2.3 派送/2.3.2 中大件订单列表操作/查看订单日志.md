订单日志用于记录中大件订单收件信息的历史修改情况。通过订单日志，用户可以查看订单中哪些字段被修改、修改前后的值、操作账号以及操作时间。

功能路径：管理端 → 订单管理 → 中大件订单列表

在需要查看的订单操作列中，点击： 更多 → 订单日志，系统将弹出【订单日志】窗口

### 订单日志字段显示
当用户修改订单的收件信息时，系统会记录本次修改内容。记录信息包括：
- 操作内容：显示修改字段、修改前后的值
- 操作人：执行修改操作的账号名称
- 日期：修改时间，格式为 `yyyy-MM-dd HH:mm:ss`

目前支持记录以下字段的修改：

**收件人信息**
- 联系人
- 联系电话
- 邮编
- 目的地
- 详细地址1
- 详细地址2
- 收件人经纬度
- 留言提醒

**派送信息**
- 派送司机
- 主批次号
- 子批次号
  
> 每个发生变化的字段都会单独生成一条日志记录。例如，同时修改联系人和联系电话时，订单日志将显示两条记录。

操作内容统一按照以下格式显示：

``` text
{字段名}由【原始值】修改为【修改后的值】
```

如果订单没有任何修改记录，订单日志窗口显示：

``` text
暂无数据
```

系统不会报错或出现异常页面。

### 常见问题
1. 为什么订单日志没有数据？

	可能有以下原因：
	-   当前订单从未修改过收件信息
	-   修改操作发生在订单日志功能上线（2026-07-09）之前
	-   修改的字段不属于订单日志记录范围

2. 一次修改多个字段会如何记录？

	每个发生变化的字段都会生成一条独立日志

3. 没有变化的字段会生成日志吗？

	不会。只有字段值发生变化时，系统才会记录日志

4. 可以修改或删除订单日志吗？

	不可以，订单日志仅用于历史追溯，不支持编辑或删除
