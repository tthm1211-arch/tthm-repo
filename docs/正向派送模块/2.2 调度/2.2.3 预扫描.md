预扫描是在正向派送业务中,由仓库库内人员在司机到场前对包裹进行的提前扫描与标记操作。通过此功能,库内人员可以在包裹上标记排单号和路线号,以提升司机到仓后的装车效率;同时为仓库交接与问题定位提供明确的记录依据。

### 预扫描节点
**司机预扫描**节点位于**包裹分拣**与**派送司机取件**之间，表示包裹已由仓库人员完成分拣，并由司机提前完成扫描确认，但尚未正式完成取件。

预扫描成功后，系统会新增一条**司机预扫描**轨迹节点，并记录对应的扫描信息。
#### 查看轨迹
功能路径：订单管理 → 中大件订单列表 → 订单列表详情 → 货物信息 → 查看轨迹

<ZoomImage src="/Pasted image 20260715164218.png" alt="批次创建" />

在轨迹信息中可查看**司机预扫描**节点及其发生时间

司机预扫描属于轨迹状态机中的中间节点，其**前置节点**必须为**包裹分拣**，仅当订单已完成仓库分拣（处于仓库收件状态）后，才允许执行预扫描操作。因此，订单轨迹不会出现从客户下单直接进入司机预扫描的情况
#### 查看扫描记录
功能路径： 订单管理→ 中大件订单列表→ 订单列表详情 → 货物信息 →扫描记录

<ZoomImage src="/Pasted image 20260715171301.png" alt="批次创建" />
**扫描账号**记录的是实际执行预扫描操作的账号，而非订单最终的派送司机。任何具有预扫描权限的司机或仓库账号，均可对处于**仓库收件**状态的包裹执行预扫描，因此扫描账号可能为任意司机账号或仓库账号，并不代表该订单最终的派送司机。
### 仓库内预扫描流程
**场景设定:** 多伦多 60 仓库安排 3 名库内人员使用各自司机账号协助进行包裹预先扫描,目的是在派送司机到场前提前识别包裹所属路线并标记排单号。

**第一件包裹 — 未分配司机:**

1. 库内人员 A 扫描第一件包裹
2. 系统提示: "单号:N2617D0000970 / 该包裹未分配司机,请联系司机管理!"
3. 库内人员 A 停止该包裹的标记操作,联系司机管理补充调度
4. 待管理端分配司机完成后,库内人员可重新扫描该包裹

**第二件包裹 — 已分配司机:**

1. 库内人员 B 扫描第二件包裹
2. 系统提示: "扫描成功 / 单号:N2617D0000971 / 路线号:99305 | 第 002 件 | 共 008 件"
3. 库内人员 B 在包裹上手写标记: "99305 #002"
4. 包裹进入预先扫描状态,等待 99305 路线派送司机接收

**第三件包裹 — 已重新分拣:**

1. 库内人员 C 扫描第三件包裹(该包裹曾被错误分配,后重新进入分拣)
2. 系统提示: "扫描成功 / 单号:N2617D0000972 / 路线号:99306 | 第 003 件 | 共 012 件"
3. 库内人员 C 在包裹上标记: "99306 #003"
4. 包裹进入预先扫描状态

**派送司机到场:**

1. 99306 路线派送司机到场,前往库内预扫描货物的存放位置
2. 司机扫描该路线下所有包裹,执行"派送扫描"
3. 扫描成功后,司机即可按排单号顺序装车出发派送
4. 整个流程中,99303、99305、99306 三条路线的所有包裹均已由库内人员完成预先扫描与标记,派送司机可直接使用标记的信息进行配送,无需重新识别
