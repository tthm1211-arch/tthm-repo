**工单列表页面**路径：管理端 -> 客诉工单 -> 工单列表
### 工单类型概述
页面中的工单分为两种类型，根据尾程派送商和跟踪单号前缀区分

| 工单类型 | 派送商 | 跟踪单号前缀 | 处理方式 |
|--------|--------|-----------|--------|
| UNI工单 | UNI | N26T1D | 在操作区点击【推送UNI】 |
| NB工单 | NB | N26XXD（XX≠T1） | 在操作列点击【处理】进入详情页回复 |

### 搜索区
支持四个搜索条件：
- **跟踪单号**：包裹的唯一标识，与包裹一对一对应
- **处理状态**：待处理 / 已完成
- **创建时间**：选择时间区间筛选
- **推送UNI**：未推送 / 已推送 / 不推送

### 操作区
**导出**
- 支持根据搜索条件导出工单数据
- 导出格式为xlsx文件，直接下载到本地，不经过【任务中心】转存
- 默认导出数据列表中的全部工单
- 当前不支持自定义导出字段，导出文件中的字段顺序与页面展示不一致，需在本地手动调整

**推送UNI**
- 将UNI工单推送回UNI进行处理
- 推送规则：跟踪单号第四位为"T"时可推送
- 推送后该工单状态自动变为【已完成】

**删除**
- 删除工单（谨慎使用，建议保留全部工单）

**隐藏搜索**
- 位置：操作区右侧（原表格视图切换位置）
- 功能：隐藏搜索区，点击再次显示或刷新页面可恢复

*本页面暂不支持表格视图切换功能

### 数据列表
#### 基础字段
- 工单单号：工单的唯一标识                                        
- 跟踪单号：订单的唯一标识，格式见[工单类型概述](#工单类型概述)                   |
- 问题类型：问题分类及子类型，格式为"类型-子类型"，来源与NB官网Support Ticket分类一致，见[附:工单问题类型完整列表](#附工单问题类型完整列表)
- 处理状态：待处理 / 已完成                                      
- 推送UNI：显示该工单是否已推送至UNI                                 
- 创建日期：Support Ticket在NB官网的提交日期                                                                  

#### 客户信息字段
- 姓名：Support Ticket中的First Name 
- 姓氏：Support Ticket中的Last Name  
- 邮箱：客户的电子邮箱地址                  
- 国家：Canada                     
- 省/州：客户所在的省份或州                  
- 城市：客户所在的城市                    

#### 工单内容字段
- 留言备注：Support Ticket中的"How can we help?"项，必填，不为空 
- 问题详情：Support Ticket中的"Details"项，非必填，可能为空 
- 附件：客户上传的图片或文件，非必填，可能为空 

#### 操作列
根据工单状态提供不同操作：
|   工单状态   |  操作   | 说明                                         |
| --------- | ----------- | ------------------------------------------ |
| 待处理  | 【处理】 | 点击进入【工单列表详情】页面，在【处理】模块中回复客户邮件。提交后状态变为【已完成】 |
| 已完成  | 【查看】 | 点击进入【工单列表详情】页面查看工单详情                       |
