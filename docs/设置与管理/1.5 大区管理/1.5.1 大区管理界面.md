**大区管理页面**路径：管理端 -> 配置中心 -> 中大件数据 -> 大区管理

### 大区及其下属概念
大区是以口岸为中心向外辐射的地理划分，整个中大件模块的分拣、路径规划、司机派送等业务流程都搭建于此框架下。
大区下的所有概念可分为两层：
1. **实体概念** -- 仓库、派送区域、司机等，具有物理地址属性，直接属于某个大区。它们与大区的关系是直接的、确定的、一对一或多对一的。例如，大区与仓库的关系是一对多：一个大区对应一个口岸分拣仓库，同时包含多个二级仓库。
2. **非实体概念** -- 路线、批次等，无独立的地址属性，通过关联的实体间接属于某个大区。一条路线属于某个大区是因为其起点或主要枢纽属于该大区，一个批次属于某个大区是因为其所在仓库属于该大区。
系统中共有5个大区：YVR（温哥华）、YYZ（多伦多）、YUL（蒙特利尔）、YYC（卡尔加里）、YEG（埃德蒙顿）。

### 搜索区

支持三个搜索条件：
1. 地区名称：按大区编码搜索
	| 大区 | 地区名称   |
    | :--- | :-: |
    | 多伦多  | YYZ  |
    | 温哥华  | YVR  |
    | 蒙特利尔 | YUL  |
    | 卡尔加里 | YYC  |
    | 埃德蒙顿 | YEG  |
2. 分拣中心：搜索已关联的分拣仓库。目前分拣中心对应的下拉菜单选项中包含了管理端中的所有仓库，因此需要在搜索框中输入关键词“Sorting Warehouse”，选中格式为”地区名称 + Sorting Warehouse"的选项
3. 启用状态：按是/否筛选

### 操作区
4.  新增按钮：创建新大区时需指定分拣中心并设置启用状态。一般情况下无需新增大区

### 数据列表
5. 表头字段：包含以下信息
	- 序号：列序号
	- 地区名称：大区编码，字段右侧有地图图标，点击弹出Google Maps窗口。如下图所示，包含以下控件：
		<ZoomImage src="/Pasted%20image%2020260617182317.png"/>
			1. 切换按钮："Map" 和 "Satellite" 两个切换按钮，用于切换地图和卫星视图
			2.  图例：显示YYC大区下四个地区的颜色及编码：Calgary-NE（红色）、Calgary-NW（绿色）、Calgary-SE（蓝色）、Calgary-SW（黑色）
			3.  全屏按钮：用于放大地图窗口
			4.  罗盘按钮，内含四向箭头指向，用于重置地图方向，点击可将地图旋转回默认北向视角
			5.  街景视图图标：点击查看该位置的街景照片
 
	- 分拣中心：与该大区关联的口岸分拣仓库，格式为”地区名称 + Sorting Warehouse”
	- 区域：鼠标悬停显示该大区下的全部派送区域名称
	- 区域数量：大该大区下的派送区域总数，区域名称见【区域】字段，区域位置见【地区名称】列中链接的Google Maps弹窗
	- 启用状态： 显示大区是否启用/停用
	- 创建时间：大区创建的时间戳
	- 操作：支持停用大区或进入区域管理页面
